PSTN Architecture & Deployment Models

US Signal supports all Webex Calling PSTN connectivity models defined in Cisco's PSTN Options framework. Our primary architecture leverages Cisco CUBE Local Gateway for enterprise flexibility, while also supporting Cisco Calling Plans and Cloud Connected PSTN (CCP) models.

PSTN Model Comparison Matrix

Cisco Calling Plans
Cloud-native PSTN provided by Cisco. Porting handled via Control Hub.
Cloud Connected PSTN (CCP)
Carrier integrated via Cisco cloud. Preferred vendors: CallTower & IntelePeer.
Local Gateway (CUBE)
On-prem SBC integration with existing carrier SIP or PRI circuits.
Dedicated Instance
Hosted CUCM with PSTN integration via CUBE or SIP trunk.
Porting Model
Cisco PSTN → Port via Control Hub.
CCP → Port through carrier.
Local Gateway → Customer retains current PSTN provider.
Redundancy
Cisco PSTN → Cloud redundancy.
CCP → Carrier redundancy.
CUBE → Dual router HA supported.

CUBE HA Dual Router Sample Configuration

interface GigabitEthernet0/0
 description WAN
 ip address 203.0.113.10 255.255.255.248
 no shut

redundancy
 mode sso
 main-cpu
  standby console enable

voice service voip
 sip
  registrar server dns:webex.com
  bind control source-interface GigabitEthernet0/0
  bind media source-interface GigabitEthernet0/0

dial-peer voice 100 voip
 destination-pattern +T
 session target dns:webex.com
 codec g711ulaw
 dtmf-relay rtp-nte
 no vad
