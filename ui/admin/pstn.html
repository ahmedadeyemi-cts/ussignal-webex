PSTN Architecture & Deployment Models

US Signal supports all Webex Calling PSTN connectivity models defined in Cisco's PSTN Options framework. Our primary architecture leverages Cisco CUBE Local Gateway for enterprise flexibility, while also supporting Cisco Calling Plans and Cloud Connected PSTN (CCP) models.

PSTN Model Comparison Matrix

Cisco Calling Plans
Cloud-native PSTN provided by Cisco. Porting handled via Control Hub.
Cloud Connected PSTN (CCP)
Carrier integrated via Cisco cloud. Preferred vendors: CallTower & IntelePeer.
Local Gateway (CUBE)
On-prem SBC integration with existing carrier SIP or PRI circuits.
Dedicated Instance
Hosted CUCM with PSTN integration via CUBE or SIP trunk.
Porting Model
Cisco PSTN → Port via Control Hub.
CCP → Port through carrier.
Local Gateway → Customer retains current PSTN provider.
Redundancy
Cisco PSTN → Cloud redundancy.
CCP → Carrier redundancy.
CUBE → Dual router HA supported.

Cloud Connected PSTN (CCP)

Primary Model

US Signal primarily leverages CallTower and IntelePeer for Cloud Connected PSTN. If a customer already has a certified CCP provider, we support that provider as well.

Full CUBE Configuration Sample

!
! ==========================================
! BASIC SYSTEM
! ==========================================
hostname CUBE-WEBEX-LGW
ip domain name ussignal.com
!
!
interface GigabitEthernet0/0
 description INSIDE-LAN
 ip address 10.10.10.10 255.255.255.0
 no shut
!
interface GigabitEthernet0/1
 description OUTSIDE-INTERNET
 ip address 203.0.113.10 255.255.255.248
 no shut
!
ip route 0.0.0.0 0.0.0.0 203.0.113.9
!
!
! ==========================================
! VOICE SERVICE GLOBAL
! ==========================================
voice service voip
 mode border-element
 allow-connections sip to sip
 ip address trusted list
  ipv4 10.10.20.5
  ipv4 203.0.113.0 255.255.255.0
 !
 sip
  bind control source-interface GigabitEthernet0/1
  bind media source-interface GigabitEthernet0/1
  transport tcp tls v1.2
  no pass-thru content sdp
  midcall-signaling passthru
  early-offer forced
  options-keepalive 30
  srtp
!
!
! ==========================================
! CODEC CLASS
! ==========================================
voice class codec 1
 codec preference 1 g711ulaw
 codec preference 2 g711alaw
!
!
! ==========================================
! TLS CONFIGURATION
! ==========================================
crypto pki trustpoint WEBEX-TP
 enrollment terminal
 subject-name CN=cube.ussignal.com
 rsakeypair WEBEX-KEY 2048
!
voice class tls-profile 1
 trustpoint WEBEX-TP
!
!
! ==========================================
! SIP UA CONFIG
! ==========================================
sip-ua
 transport tcp tls v1.2
 registrar dns:customer.lgw.webex.com expires 3600
 sip-server dns:customer.lgw.webex.com
 connection-reuse
 authentication username example@ussignal.com password 0 MySecurePassword realm ussignal.com
 crypto signaling default trustpoint WEBEX-TP
!
!
! ==========================================
! SIP PROFILE (Optional Header Fix)
! ==========================================
voice class sip-profiles 1
 request INVITE sip-header Contact modify "" ""
!
!
! ==========================================
! TRANSLATION RULES (E.164)
! ==========================================
voice translation-rule 1
 rule 1 /^9\(1[2-9]..[2-9]......\)$/ /+\1/
!
voice translation-profile TO-WEBEX
 translate called 1
!
!
! ==========================================
! DIAL-PEER TO WEBEX (OUTBOUND)
! ==========================================
dial-peer voice 100 voip
 description OUTBOUND TO WEBEX
 destination-pattern +T
 session protocol sipv2
 session target dns:customer.lgw.webex.com
 voice-class codec 1
 voice-class sip tls-profile 1
 voice-class sip profiles 1
 voice-class sip options-keepalive up-interval 30 down-interval 30 retry 3
 translation-profile outgoing TO-WEBEX
 transport tcp tls
 dtmf-relay rtp-nte
 srtp
 no vad
!
